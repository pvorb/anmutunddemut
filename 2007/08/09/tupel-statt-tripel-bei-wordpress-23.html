Tupel statt Tripel bei Wordpress 2.3

Wie gestern berichtet, kommt Wordpress in der nächsten Version mit einem überarbeiteten und vereinheitlichtem Kategorie/Linkstypen/Tag System um die Ecke. Weil es eine direkte Konkurrenz für mein Semantic Web Plugin ist, hat es mich nicht losgelassen - im Zweifel muß man ja mit dem großen Haien schwimmen und migrieren. Also wollte ich wissen, wie genau, das implementiert werden soll. Im aktuelle Development-Snapshot [download] kann man sich anschauen, wie das aussehen wird:


CREATE TABLE $wpdb->terms (
term_id bigint(20) NOT NULL auto_increment,
name varchar(55) NOT NULL default '',
slug varchar(200) NOT NULL default '',
term_group bigint(10) NOT NULL default 0,
PRIMARY KEY (term_id),
UNIQUE KEY slug (slug)
) $charset_collate;


CREATE TABLE $wpdb->term_taxonomy (
term_taxonomy_id bigint(20) NOT NULL auto_increment,
term_id bigint(20) NOT NULL default 0,
taxonomy varchar(32) NOT NULL default '',
description longtext NOT NULL,
parent bigint(20) NOT NULL default 0,
count bigint(20) NOT NULL default 0,
PRIMARY KEY (term_taxonomy_id),
UNIQUE KEY term_id_taxonomy (term_id,taxonomy)
) $charset_collate;


CREATE TABLE $wpdb->term_relationships (
object_id bigint(20) NOT NULL default 0,
term_taxonomy_id bigint(20) NOT NULL default 0,
PRIMARY KEY (object_id,term_taxonomy_id),
KEY term_taxonomy_id (term_taxonomy_id)
) $charset_collate;

Der wichtige - um nicht zu sagen verherrende - Teil ist die letzte Tabelle. Die Beziehung zwischen Post/Pages/Links auf der einen Seite und Kategorien und Tags auf der anderen Seite wird in einer zweispaltigen Tabelle gemacht. Tupel. Damit mach Wordpress den gleichen Fehler wie Drupal und fällt hinter das, was sie selber mit den Post-Meta-Daten (die schon als Tripel angelegt sind) schon im System an Potential haben zurück. Traurig.

Zugegeben. Weder Wordpress noch 99% der Nutzer brauchen derzeit mehr als Tupel. Aber es ist ja nicht so als wären RDF und Tripel eine Idee, die gestern erst geboren wurde. Und in der Implementierung macht es kaum einen Unterschied. Wordpress hätte sich mit einer Entscheidung für ein tripelbasiertes Metadatensystem soviel Potential erschlossen. Es wäre soviel angenehmer geworden, für Entwickler, saubere Plugins zu schreiben. Und im Ganzen wäre die Anwendung soviel zukunftssicherer. Das ist so ärgerlich, dass ich denen fast eine Mail schreiben mag. Damnit.

Nachtrag: also es ist mir wirklich ein Rätsel, warum sie bei der Neugestaltung des Metadaten-Architektur nicht auf das Post-Metadata-System zurückgreifen. Immerhin verwendet Wordpress selber die Tabelle schon für eine Reihe von Funktionen. Unbegreiflich.
