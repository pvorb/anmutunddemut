Semantic Weblog Plugin for Wordpress 2.x

The Plugin makes use of the Post-Metadata system to store more Semantic Data. In the End it aims to enable Wordpress to be a form of extreme lightweight Ontology Editor and Ontology. The fact that the Wordpress Post-Metadata-System is created as a Triplestore, allows to create typed semantic Statements between Nodes (Posts/Pages) and Strings (pure Strings or URLs) or other Nodes (Posts/Pages).

1. Setup

  1. Be sure to have the AAAWP Plugin setup and running before you setup the Semantic Weblog!
  2. The most important part of the plugin is already installed, the fantatsic Wordpress Post-Meta-Data-Triple-Store.

    So it does not need any further Database Tables.
  3. The Plugin finds the Metadata which are meant to be used with it, via a Prefix within the key Column of the Post-Metadata-Table, which has to be added to the Key.

    The default Prefix is “sw_”. You may change this prefix within the class “semanticweblog” within function “__construct”.

    A configuration possibility within the Worpress Backend will be added soon!

2. Add Metadata to Posts

To add Metadata to a Posts, just use your Prefix and the desired semantic Key and the desired Value.

You can do this even without the Plugin activated. This is quite beautifull as you can use the data gathered later maybe in a different way without having to migrate them.

Expample 1:

  1. You want to state that a Post is in the Movie “Star Wars”.
  2. Within the “Create New Post”-Form or the “Edit Post”-Form you scroll down to the “Custom Fields” Box below the Textarea.
  3. You enter “sw_movie” into the “Key” input field, and “Star Wars” into the “Value” input field.

    (NOTICE: if you chose “sw_” as Key-Prefix. see 1. Setup. If you changed the default prefix, use that one.)
  4. You press the “Add custom field” button.
  5. Done

If you have used a key before, you can look if it appears in the Key-Selection-Form.

Wordpress by default does only add about 20 keys into the dropdown. So Keys you have

already used may not appeare there. But don’t worry. Just write the in the input form.

3. Display Metadata in the Template

Right now there is no function to display Metadata automatically by this plugin within any Theme. So you have to integrate it by yourself. But as the Access to the Metadata is part of the Wordpress Core it is not very difficult.

Notice: I’ll be adding more functions (e.g. “Template Tags”) soon. Right now, you can use all functions which come with the module. But do yourself a favour and don’t try to call the Methods defined within den Semanticweblog Class from a tempalte.

Within the Loop you can use the following line to get all Custom fields

$customs = get_post_custom( $post_id = 0 );

The result is a multidimensional array, which may look like this:

Array

    [sw_Director] => Array
        (
            [0] => Lucas, George
            [1] => Rottenberry, Gene
        )

    [sw_Actor] => Array
        (
            [0] => Ford, Harrison
            [1] => Shattner, William
        )

    [sw_Movie] => Array
        (
            [0] => Star Wars
            [1] => Star Trek
        )

    [sw_Keyword] => Array
        (
            [0] => Science Fiction
            [1] => Spaceships
        )

)

And you can iterate like this:

// Iterate on the keys
foreach ($customs as $key=>$values){
	// Different ifs for groups of Keys:
	if( $key == "sw_Actor" or $key == "sw_Director"){
	  // Display Name of the Key
		?><h3><?=sw_clean_key($key)?></h3>
		<ul>
		<?
		// Iterate on the single Metadata per Key
		foreach ($values as $value){
			// Display single Metadata
			?><li><?=$value?></li><?
		}
		?>

		</ul>
		<?
	// Further keys
	}elseif($key == "sw_Movie"){
		...
	}else{
		// default Keys
	}

}

4. Create Register-Pages

To Create Pages with gather together all the precious Metadata you’ve gathered, you only have to do a few things.

  1. Doublicate you Template for static Pages and rename it to “register.php”. You may choose another name, if you like. And you’ll have to do this step only once, for all Register-Pages.
  2. Change the “Template Name”-Comment within the Template to /* Template Name: Register */
  3. Add these Lines within your Post-Text-div:
    <?sw_theme_letters($_GET['letter']);?>
<?sw_theme_register($post_id, $_GET["letter"]);?>

    And make sure, they are within "The Loop"!

  4. Create a static Page
  5. Assign the “Register”-Template to that page
  6. Add as Customdata for each key you have used in you regular Posts a Metadata in which the key is “sw_indexfor” and the value is the key you would like this page to be the register for. For Example:
    • Within you posts you have the Realtions “sw_Actor” and “sw_Director”
    • You want to have a register for both of them, and you already created a Page “People” and done the steps above
    • Add the custom field with the key “sw_indexfor” and the value “sw_Actor”.
    • Add the custom field with the key “sw_indexfor” and the value “sw_Director”
  7. Done

With that Mechanism, you can create several Registers each for as many Relation-Types you want. Of course can even use one Relations-Type in more than one Register.

I suggest to have at least one Register, which is the Index for all Relation-Types you use.
