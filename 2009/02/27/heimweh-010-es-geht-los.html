Heimweh 0.1.0 – Es geht los!

Yeah! Voran! Mit ein bisschen Starthilfe von Patrick sind jetzt die ersten zaghaften Zeilen Code im Github. Gestern im Zug hatte ich ja, wie schon erwähnt mit dem Menu-Router experimentiert. Jetzt stehen langsam Entscheidungen an. PHP ist inzwischen gesetzt. Damit kenne ich mich zu gut aus, um jetzt was Neues auszuprobieren.

Zwei Fragen die sich jetzt stellen sind folgende.

Wie die XML-Dokumente speichern?

Defakto bieten sich drei Möglichkeiten an.
1. Eine einzige Datei ins Filesystem?
Dafür spricht, dass ich damit prima suchen/ersetzen machen kann.
Dagegen spricht, dass das evtl. schnell an die Grenze der Performanz stoßen könnte und das Risiko die ganze Datei beim Schreiben zu zersägen groß ist.
2. Eine Datei pro Dokument ins Filesystem?
Dafür spricht, dass ich den Editor dann auch leicht für alle anderen möglichen Systemdateien gebrauchen kann und keine Datenbank brauche.
Dagegen spricht, dass ich dieser Art der Datenhaltung nicht traue. Ich bekomme schlicht nicht mit, wenn eine Datei verschwindet. (Hatte ich in meinem alten System schonmal.)
3. Eine minimalistische Datenbank? Eine Tabelle. Ein Zeile pro Dokument. Drei Spalten: ID, Datum, XML-Quelltext.
Dafür spricht, dass ich der Datenhaltung traue, die grundsätzlich performant ist und ich zur Not auch noch über die ganze Tabelle suchen kann (das Datum ist schon ein Eingständnis in diese Richtung.)
Dagegen spricht, dass es eigentlich nicht sooo schön ist.

Wie den Code verteilen?

Alles PHP in eine Datei?
Eine Datei pro Klasse?
Überhaupt: Objektorientiertes PHP?
Einen Ordner für alle Module?
Und darin nochmal einen Ordner pro Modul?
Einen Ordner für alle alle Themes und darin einen Ordner pro Theme?

Any Opinions?

Soviele tolle Entscheidungen, die gefällt werden wollen … das macht Spaß!
