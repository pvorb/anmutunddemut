Github.com

Basti – of the mighty mighty Yolk – hat mir Github neulich bei einem Whisky nahegelegt. Ich war über diesen oder jenen Weg im Netz die Tage schon mal drauf gestoßen. Aber wie das immer so ist. Erst wenn man was von jemanden empfohlen bekommt, auf desen Meinung man Wert legt, gibt man den Dingen eine echte Chance. Zumal unter dem Einfluß eines Bowmore, kicher.

Github ist im Grund ganz ähnlich wie Assembla: Eine online Versionierungsplatform mit ein paar netten Features drum herum und besonders dazu angetan, kollaborativ zu arbeiten. Bisher nutze ich eigentlich nur Gist, eine Art Abfall-Produkt, das v.a. dazu gut ist, kleine Code-Schnipsel online zu stellen, so dass man die mal wem vorzeigen kann. Sowie mein Itunes-Mediathek-Testskript. Nett dabei: Man kann die Dinger auch später wieder bearbeiten und Git legt gleich Revisionen an.

Eigentlich finde ich den ganzen Vorgang extrem sexy. Basti hat auch betont, wie wichtig die soziale Komponente bei Github ist. Jeder kann den Code des anderen gleich sehen, forken und dem ursprünglichen Autor direkt als systemimmanenten Patch wieder anbieten. Schlußendlich scheint mir das in einer Art Social Software Development online Plattform zu münden. Eine Facebookwikipedia des Codes. Open Source Produkte haben ja jetzt schon eine hohe Eigendynamik. Wie das erst wird wenn sich das mit dem Komfort und der Explosionskraft moderner sozialer Platformen mischt. Wie sollen kommerzielle Produkte da noch mithalten?

Und obendrein es mündet in der Verlagerung der IDE in eine Webapplikation. Tschüss Zend Studio. Hallo Github. Leider ist es wohl noch nicht ganzsoweit. Das muss ich mir mal alles noch von Basti erklären lassen. In Vorbereitung auf mein eigenes neus CMS werde ich mich da jetzt aber mal selber ein bisschen reinfuchsen.

P.S.: Bin da als Ben the Bear (ihr wißt schon, der Bär von Grizzly Adams) unterwegs.
