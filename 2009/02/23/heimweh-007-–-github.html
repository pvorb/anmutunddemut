Heimweh 0.0.7 – Github

Ich geb es gerne zu: Ich bin schmutzig. Als PHPler wächst man quasie im Schmutz auf. Ich habe ja die Hoffnung, dass das – wie es auch von Kindern behauptet wird – das Immunsystem stärke. Aber ich liebe es nunmal Code, direkt auf dem Live-Server in der Live-Installation zu entwickeln. Kein Staging, kein Testing. Auf der Arbeit kann ich mir das nicht mehr erlauben. In meinem Blog darf ich das aber. Trotzdem bin ich ziemlich begeistert von Github. Und zwar nicht weil ich nun Einsicht in die Großartigkeit von Versionierung und Staging hätte, sondern, weil ich die Transparenz und soziale Dynamik geil finde. Jeder kann sofort jede Zeile Code sehen, runterladen, verändern, mir wieder anbieten. Das klingt nach Spaß.

Ich bin noch am Rätseln, wie ich das hinbekommen kann, aus dem Spaß nicht zuviel Arbeit werden zu lassen. Am besten gar keine Arbeit. Irgendwie müsste meine index.php sich immer gleich als erstes mal ein aktuelles Update von Githup auf den Server ziehen. Dann könnte ich in/mit Github entwickeln und auf dem Liveserver wären immer sofort die richtig Daten. Das wäre buper.

Der Compiling and installing git on Mac OS X Guide sieht für mich alten Shellphobiker alledings schon reichlich abschreckend aus. Ich ziehe bereits in Betracht, alles in einem Gist zu machen, weil ich den so schön online in einer Textarea bearbeiten kann.
