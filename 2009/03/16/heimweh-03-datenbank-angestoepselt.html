Heimweh 0.3 – Datenbank angestöpselt

Habe heute die Tabellen und die Datenbank angestöpselt. Da ich wenig Ahnung von Datenbankoptimierung habe, lesen sich die Create Table Statements für Kenner und Könner wohl eher amüsant. Mir reichen sie aber für's Erste und damit ist die Hauptbedingung erfüllt. Gerne nehem ich latürnich sachdienliche Hinweise entgegen.

 CREATE TABLE `heimweh_documents` (
`id` INT NOT NULL ,
`date` DATETIME NOT NULL ,
`xml` LONGTEXT NOT NULL ,
PRIMARY KEY ( `id` ) ,
INDEX ( `date` ) ,
FULLTEXT (
`xml`
)
) ENGINE = MYISAM ;


 CREATE TABLE `heimweh_metadata` (
`mid` INT NOT NULL AUTO_INCREMENT PRIMARY KEY ,
`subject` TEXT NOT NULL ,
`predicate` TEXT NOT NULL ,
`object` LONGTEXT NOT NULL ,
FULLTEXT (
`subject` ,
`predicate` ,
`object`
)
) ENGINE = MYISAM ;

Was noch nicht so ganz unerwähnt bleiben soll ist folgendes. Heimweh unterstützt Mysql. Mehr nicht. Dementsprechend gibt auch keinen Datenbankabstraktionslayer und keine Wrapper-Klasse. Die Datenbank wird in der index.php angezapft und danach in den Modulen einfach mit den mysql_query und mysql_fetch_x Funktionen angesprochen. Fertig. Weniger ist mehr.
