Drupal Tricks – Schema API

Ein geiles Feature in Drupal ist die Schema API. Man definiert die Datenbankstrukturen zu einem Modul in einem Array, ebenso wie Ergänzungen und Veränderungen in späteren Versionen des Moduls. So kann man Module sauber installieren, updaten und deinstallieren. Das ist nicht ganz so professionell wie ein Objektpersistenter Datenbanklayer, aber immerhin professioneller, als Create-Table Anweisungen irgendwo im Modul-Code.

Leider muss man dafür eigentlich wissen, wie man eine gewünschte Datenbankstruktur richtig beschreibt. Meistens hab ich einfach Schemas anderen Module kopiert und angepasst. Auf dem Wege hab ich mir mal meine Teaserimage-Tabelle zersägt, was schon zeigt, dass das nicht ganz der richtig Weg ist.

Fuchsiger ist folgendes: Man installiert und aktiviert das Schema Modul. Dann legt man die Tabellen, die man gerne hätte mit dem phpMyAdmin an. Dann sucht man sich unter admin/build/schema/inspect das Schema-Array raus und kopiert es in die .install Datei des Moduls. Dann die Tabellen wieder löschen und das Modul installieren. Fertig.
