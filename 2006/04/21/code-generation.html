Code Generation

Mein Physik-LK-Lehrer war ein schlechter Lehrer aber zu gleichen Teilen ein guter Entertainer und Dozent. So stammt der schöne Spruch "Ja, da legen Sie ihr Hirn in Falten." von ihm. (Besonderns schöne Sitte an meiner Schule: die Schüler wurden von den Lehrer ab einem gewissen Punkt beim Vornamen Gesiezt. "Benjamin, halten Sie mal den Tauchsieder!") In den letzten Tage ist ist mir der eine oder andere Spruch vom guten Auf der Heide wieder in den Kopf gekommen, denn die Inhalte des Leistungskurses Physik zieren sich mitunter sich in den Falten des Hirns zu legen und ähnliche Verrenkungen im Hirn ergeben sich, wenn man anfängt Code mit Code zu generieren.

Als erstes ist mal das Debugging fusselig, aber daran kann man sich ja gewöhnen. Was aber schon mal ein paar Tage braucht, sind die ulkigen Verknotungen, die sich im Hirn ergeben, wenn man die Klassen, die die zu erzeugenden Klassen beschreiben mit den Ideen dahinter vermischt. Überhaupt vermischt sich alles miteinander. Klar. Wenn mann sich in Ruhe hinsetzt, dann ist das alles logisch, aber so im Eifer des Gefechts, ist ein Klasse, die Instanzen einer anderen Klasse beschreibt schon mal was anderes - rosa Kaninchen und so. Ach, mein Gesabbel versteht ja eh niemand.

Wichtige Einsicht nur hier: ich bin inzwischen zum dritten oder vierten Mal in meiner Entwickler-Entwicklung an eine Stelle gekommen, an der es notwendig wurde, Code mit Code zu erzeugen. Und ich kann sagen: wenn es erstmal läuft, dann ist es gar nicht so schwer und ganz zauberhaft, ganz zauberhaft mächtig vor Allem. Unterschätze niemals die Macht der Abstraktion! Abstraktion ist die Ultima Ratio der Programmierung, das zeigt sich immer wieder. Ich hätte gut Lust Programmier-Theoretiker zu werden. Und langsam hab ich das ungute Gefühl, dass das ordentlicher gehen müsste, mit der Code Generierung. Da sollte es saubere Konzepte für geben, bitte!

"Herr Ober! Einmal Code au Vin, bitte."
