o:XML

o:XML ist krasses Zeug! o:XML ist eine ObjektOrientierte Programmiersprache, die in XML geschrieben wird ähnlich wie XSL. Das ist zwar nicht übermäßig praktisch weil man soviel schrieben muss und sich viel vertippen kann, aber mit dem richtigen Editor
kann man solche Programmiersprachen eigentlich ganz gut schreiben.
o:XML versteht sich aber als ganzheitliches XML erzeugt o:XML nicht nur standardmäßig XML-Output, sondern ist in JEDES beliebige XML-Dokument einfügbar. Das funtioniert, deshalb, weil o:XML serverseitig interpretiert wird. Beim Client kommt dann sauberes HTML, SVG oder was auch immer an. Dass o:XML auf dem Server ausgeführt wird hat zudem den gewaltigen Vorteil, dass o:XML auf Daten auf dem Server zugreifen kann und auch für die Arbeit auf dem Dateisystem spezielle Funktionen hat. Das ist ein klarer Pluspunkt für o:XML gegnüber XSL.

o:XML verwendet für die Referenzierung seiner Objekte, deren Eigenschaften und Methoden zudem eine Syntax, die XPath entlehnt ist und die beliebige Internet-Resoucen miteinschließt. Damit kann o:XML zum Beispiel anderer Leuts RSS-Feeds einlesen und auf einer Seite als HTML oder SVG darstellen.

Ein Vorteil, der, wenn man ih zuende denk einem das Hirn verbiegt, ist der, dass XML-basierte Programmiersprachen mit XSLT in andere Programmiersprachen transformiert werden können. So könnte man o:XML in PHP transformieren. Aber wie gesagt, da will ich lieber nicht drüber nachdenken.

Leider hat o:XML für mich z.Zt. einen gravierenden Nachteil: Es ist in Java programmiert, was bedeutet, dass man es irgendwie kompliziert auf dem [am besten eigenen Server] installieren muss, und dann wohl nicht sooo performant ist.

Ansonsten ist o:XML aber fantastisch, ich würde mich sofort von PHP scheiden lassen und umgehend mit o:XML zu Standesamt rennen. Gebt mir o:XML!
